# CareerCollection — Agent Instructions

## 使用前必读 / 開始前に必ず読む

**每次任务开始前，先读取 `profile.md`，获取用户的学历・职历・语言・目标信息。**
**毎回タスク開始前に `profile.md` を読み込み、ユーザーの学歴・職歴・言語・ターゲット情報を把握する。**

---

## 文件结构 / ファイル構成

```
CLAUDE.md                    ← 本文件（Agent Instructions）
profile.md                   ← 求职者个人信息（每次必读）
README.md                    ← 企业总览索引
tracker.md                   ← 求职进度追踪
companies/_template.md       ← 企业信息模板
companies/[企業名].md        ← 各企业详细信息
export_excel.py              ← Excel导出脚本（按需运行）
```

---

## Workflow 1 — 新增企业 / 企業を新規追加する

**触发条件**：用户请求调查或新增某家企业时。

**执行步骤**：
1. 读取 `profile.md` 确认用户背景
2. 通过 WebSearch / WebFetch 调查以下内容：
   - 基本信息（成立时间・总部・上市情况・员工数・营收）
   - 近况与动态（近1〜2年的新闻）
   - 新卒招募情况・岗位・年龄限制
   - 中途招募中是否有 BD/PM/营销类岗位
   - 企业文化・工作语言・对外籍人才的态度
   - 招募页面 URL
3. 根据 `profile.md` 中的「応募可能ルート」判断哪些路线有效
4. 复制 `companies/_template.md`，创建 `companies/[企業名].md`
5. 在 `README.md` 的企业列表中追加（按 Tier 分类）
6. 在 `tracker.md` 中追加该企业

**Tier 判断基准**：
- **Tier A**：有效路线 2 条以上，现阶段可以立刻行动
- **Tier B**：有 1〜2 条有效路线，存在竞争率・文化・时机等障壁但可突破
- **Tier C**：求人依赖型，有空缺时立刻行动
- **Tier D**：中长期目标，现阶段经验不足，1〜2年后再挑战
- **リストから除外**：当前没有可投岗位的企业，不写入 README，不创建 MD 文件

## Workflow 2 — 更新求职进度 / 求職進捗を更新する

**触发条件**：用户用自然语言汇报进度时。
（例：「○○投完了」「○○面试通过」「○○不投了」）

**执行步骤**：
1. 打开对应企业的 `companies/[企業名].md`
2. 更新 `応募ステータス` 部分（状态・投递日期・选考日程）
3. 更新 `tracker.md` 中该企业的对应行
4. 若状态变为 **「応募済」及之后** → **自动执行 Workflow 3**
5. Excel 仅在用户明确要求时才运行 `export_excel.py` 生成

---

## Workflow 3 — 面试准备 / 面接準備を行う

**触发条件**：企业状态变为「応募済」「選考中」时自动执行。

**执行步骤**：
1. 从以下渠道收集面试经验：
   - OpenWork（openwork.jp）
   - 就活会議（syukatsu-kaigi.com）
   - Glassdoor
   - X（Twitter）口碑
   - 各公司招募页面（重新确认求人人物像）
2. 将调查结果填入 `面接体験談（口コミ収集）` 部分：
   - 选考流程（几轮面试）
   - 常见问题
   - 企业重视的要素
   - 注意事项与易踩坑的地方
3. 结合 `profile.md` 中的用户画像制作 `面接プラン`：
   - 志望动机的核心逻辑（该企业 × 用户强项的结合点）
   - 重点突出的优势优先顺序
   - 常见问题的参考答案
   - 反问问题列表

---

## Workflow 4 — 定期更新企业信息 / 定期的な企業情報の更新

**触发条件**：用户要求「更新信息」时。

**执行步骤**：
1. 通过 WebSearch 获取指定企业（或全部企业）的最新新闻
2. 更新 `近況・動向` 部分
3. 确认新卒・中途的最新招募情况并更新
4. 若有变化，重新审视 `README.md` 中的优先度与状态

---

## 常时适用的规则 / 常時適用するルール

- 每次任务开始前先读 `profile.md`，不依赖记忆中的用户信息
- 信息可信度低时，注明「参考値」「要確認」
- 招募信息与采用页面 URL 必须记载
- 企业文化评价以口碑（OpenWork・Glassdoor）为依据
- 适配度只从 `profile.md` 中的目标职种视角判断
- 新卒岗位的年龄申请风险每次必须明确提示
- Excel 仅按需生成，不自动生成
